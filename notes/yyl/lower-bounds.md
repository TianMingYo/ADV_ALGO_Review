# 下界部分笔记

## 阅读范围

本篇记录第 5-6 节，主要关注论文的两个复杂度模型，以及 distance order problem 的时间和比较下界。

## 两个模型

第 5 节定义了 comparison model 和 time model。

comparison model 中，算法已知图结构，可以通过 oracle 访问边权，代价主要是比较边权线性函数。这个模型更抽象，也更强。

time model 中，图以邻接表给出，算法必须遍历邻接表才能看到弧和弧长。因此访问输入本身也要计入时间。这个模型更接近实际输入。

## 时间下界

Lemma 6.1 证明任何正确算法在 time model 中需要 `Ω(m)` 时间。直觉是：如果算法没有查看某些邻接表或某些弧，就可以构造一条未被看到的短边，使输出顺序不再正确。

这个下界说明，即使某些图在比较意义下很简单，输入访问成本也无法完全避免。

## `log D` 比较下界

Lemma 6.2 使用决策树思路。图可能有 `D` 个 distance order。对每个 distance order，可以选择一组边权使其成为唯一 true distance order。

正确算法必须区分这些可能性，因此最坏情况下至少需要 `Ω(log D)` 次比较。这里的 `D` 反映图结构允许多少种合法距离顺序。图越“自由”，可行顺序越多，算法需要的信息量越大。

## `F - n + 1` 比较下界

Lemma 6.4 证明还需要 `Ω(F - n + 1)` 次比较。证明直觉是固定一个前向弧最多的 distance order，并给前向弧设置长度。

如果算法比较太少，就存在扰动前向弧长度的自由度，使算法沿同一决策路径运行，但原顺序已经不再是 true distance order。

这里 `n - 1` 对应保证每个非源点可达的基本树边，剩余的前向弧才是必须通过比较排除或确认的额外信息。

## 合并结论

Theorem 6.5 给出 time model 下界：

`Ω(m + log D)`。

Theorem 6.6 给出 comparison model 下界：

`Ω(F - n + 1 + log D)`。

无向图中对应为：

`Ω(m - n + 1 + log D)`。

这些下界为后面证明 Dijkstra 及其扩展版本的 universal optimality 提供了目标。
